# 프론트엔드 기능 명세서

> 하프피프티 · 계약서 위험 안내 서비스
> 담당: 김민찬 · 스택: React 18 + TypeScript + Vite + TailwindCSS
> 기준 브랜치: `feat/frontend-uiux-redesign` (리디자인 버전)

---

## 1. 개요

계약서(PDF 또는 텍스트)를 입력받아 조항별 위험 분석 결과를 사용자 눈높이에 맞춰 보여주는 SPA.
회원가입 없이 사용하며, 8개 화면의 단방향 플로우로 구성된다.

```
랜딩 → 업로드 → 추출 확인 → 맞춤 설정 → 분석 진행 → 결과 요약 → 조항 상세 → 결과 활용/삭제
```

---

## 2. 화면별 명세

### 2-1. 랜딩 (Landing)

| 항목 | 내용 |
| --- | --- |
| 목적 | 서비스 가치 전달, 신뢰 확보, 분석 시작 유도 |
| 구성 | 히어로(카피 + 분석 미리보기 목업), 기능 소개 3종, 이용 방법 3단계, 개인정보 안내 섹션, 푸터 |
| 액션 | `분석 시작하기` CTA 2곳 → 업로드 화면 이동 |
| 비고 | 푸터에 "법률 자문 아님" 고지 필수 |

### 2-2. 계약서 업로드 (Upload)

| 항목 | 내용 |
| --- | --- |
| 목적 | PDF 업로드 또는 텍스트 붙여넣기 입력 |
| 입력 방식 | 탭 전환: PDF 업로드 / 직접 붙여넣기 |
| PDF 검증 | 확장자·MIME(`application/pdf`), 최대 10MB — 실패 시 인라인 에러 |
| 텍스트 검증 | 30자 이상 |
| 인터랙션 | 드래그앤드롭 지원(드래그 중 하이라이트), 파일 선택 후 파일명·용량 표시, 재선택 가능 |
| 동의 | 개인정보 처리 동의 체크박스 — 체크 전 다음 버튼 비활성 |
| 다음 버튼 | 조건 미충족 시 비활성 + 사유 문구 표시 ("계약서를 올려주세요" 등) |

### 2-3. 추출 결과 확인 (Extract)

| 항목 | 내용 |
| --- | --- |
| 목적 | 텍스트 추출이 제대로 됐는지 사용자 확인 (스캔 PDF 예외 분기) |
| 구성 | 추출 본문(스크롤 영역) + 사이드: 문서 정보(문서 종류/조항 수/인식률), 상태 배너 |
| 액션 | `이 내용으로 계속` / `다시 올리기` |
| TODO | 현재 본문은 목업 — 에이전트 추출 API 연동 후 실제 추출 텍스트 표시 필요 |

### 2-4. 사용자 맞춤 설정 (Persona)

| 항목 | 내용 |
| --- | --- |
| 페르소나 | 일반 성인 / 고령층 카드 선택 (단일 선택, API `persona` 파라미터로 전달) |
| 접근성 토글 | ① 글자 크게(전역 rem 스케일 16→18px) ② 높은 명암(흐린 텍스트 진하게) ③ 음성 안내(조항 설명 TTS) |
| 원칙 | 위험 판단 기준은 동일, 설명 말투·난이도만 변경됨을 명시 |
| 액션 | `계약서 분석 시작` → 분석 API 호출 + 진행 화면 이동 |

### 2-5. 분석 진행 (Progress)

| 항목 | 내용 |
| --- | --- |
| 목적 | 수십 초 걸리는 분석 대기 UX |
| 구성 | 스피너, 진행 바(로딩 중 92%까지 증가, 완료 시 100%), 4단계 체크리스트(읽기→조항 분리→위험 탐지→품질 확인) |
| 성공 시 | 0.9초 후 결과 요약으로 자동 이동 |
| 실패 시 | 에러 메시지 + `다시 시도` / `예시 결과 보기` 분기 |
| 액션 | 로딩 중 `분석 취소` 가능 |

### 2-6. 분석 결과 요약 (Summary)

| 항목 | 내용 |
| --- | --- |
| 헤드라인 | "전체 N개 조항 중 확인이 필요한 조항이 M개" (위험+주의 합산) |
| 통계 | 위험/주의/안전 카운트 카드 3종 |
| 최우선 확인 | 첫 번째 위험 조항 하이라이트 배너 → 클릭 시 상세 이동 |
| 필터 | 전체/위험/주의/안전 — 실시간 필터링 동작 |
| 조항 카드 | 위험 유형, 설명(2줄 말줄임), 근거, 위험도 배지 → 클릭 시 상세 이동 |
| 유틸 | `질문 전체 복사` (전 조항 확인 질문 일괄 복사) |
| 예시 모드 | 백엔드 미연결 시 샘플 데이터 표시 + 상단 경고 배너 |

### 2-7. 조항 상세 (Detail)

| 항목 | 내용 |
| --- | --- |
| 레이아웃 | 좌측 조항 목록(위험도 점 표시, 클릭 전환) + 우측 상세 |
| 상세 구성 | 위험 배지 → 쉬운 설명 → 왜 확인해야 하나(근거) → 계약서 원문(복사 가능) → 확인 질문 목록(개별 복사) |
| 음성 안내 | 토글 ON 시 화면 진입 때 설명을 `speechSynthesis`(ko-KR)로 재생 |
| 액션 | `결과 요약으로` / `확인 끝내고 결과 활용하기` |

### 2-8. 결과 활용·삭제 (Done)

| 항목 | 내용 |
| --- | --- |
| 액션 카드 | ① 확인 질문 일괄 복사 ② 인쇄·PDF 저장(`window.print`) ③ 전문가 상담용 요약 복사(위험·주의 조항 + 원문) |
| 데이터 삭제 | "24시간 뒤 자동 삭제" 안내 + `지금 모두 삭제`(2단계 확인) → 삭제 완료 화면 |
| TODO | 삭제 버튼을 백엔드 `DELETE /api/contracts/{id}` 와 연동 필요 (현재 UI만 동작) |

## 4. API 연동

| 함수 | 호출 대상 | 비고 |
| --- | --- | --- |
| `analyzeContract(text, persona)` | `POST {백엔드}/api/contracts/analyze` | 정상 경로 |
| `analyzePdf(file, persona)` | `POST {에이전트}/analyze-pdf` | **임시** — 백엔드 프록시 완성 후 `POST {백엔드}/api/contracts/analyze-pdf` 로 교체 필요 |

에러 처리: 422(스캔 PDF), 415(PDF 아님), 그 외 상태코드 → 사용자 문구로 변환해 진행 화면에서 분기.
