# 재현성(Reproducibility) 검증

**목적**: `temperature=0`인데도 실행마다 오탐 개수·Judge 점수가 흔들린 정황이 관찰됐다.
같은 입력을 5회씩 반복 실행해 편차를 수치화하고, 보고서에 결과 수치를 쓸 때 감안해야 할
±편차를 명시한다.

**대상**: 확정 위험 조항 4건(광고대행, 신발도매, 즉시연금 2017-17, 즉시연금 2018-8) +
즉시연금 2018-8의 쟁점 문구를 살린 진단용 변형(b) + 정상 예금거래기본약관, 총 6개
문서 × 5회 = **30회 실행**. 모두 `persona="adult"`, Ours(4단계 파이프라인,
`run_pipeline`), 완전히 동일한 입력 텍스트.

**즉시연금 2018-8 두 버전**:
- **(a) 원본 그대로** — Parser가 별표1(쟁점 문구 "만기보험금을 고려하여" 포함)을 "첫 조항
  앞 전문"으로 판단해 버리는 상태. 이 잘림 자체가 5회 내내 재현적으로 일어나는지 확인용.
- **(b) 쟁점 문구를 살린 진단용 변형** — 별표1 앞에 인식 가능한 조 번호만 붙여 Parser가
  버리지 않게 한 버전(원문 문구는 그대로, `feat/baseline-eval`에서 쓴 것과 동일한 텍스트).
  쟁점이 입력에 있을 때 판정·근거가 회차마다 흔들리는지 확인용.

### 정상 예금거래기본약관 (24개 조항)

23/24개 조항은 5회 내내 완전히 동일(안전/해당없음, 또는 제19조만 위험/책임 면제×5).
**유일하게 흔들린 조항**:

| 조항 | 1회차 | 2회차 | 3회차 | 4회차 | 5회차 |
|---|---|---|---|---|---|
| 제15조(clause_015) | 안전/해당없음 | **주의/책임 면제** | 안전/해당없음 | **주의/일방적 계약 해지** | 안전/해당없음 |

문서 전체 Judge 평균은 5회 모두 4.0으로 동일했다(조항 24개 평균이라 개별 조항의 흔들림이
평균에 거의 묻힘).

## 2. 판정이 일관된 조항 / 흔들린 조항

- **5회 내내 완전히 일관됨(risk_level·risk_type 둘 다 100%)**: 위험 조항 4건 전부(광고대행,
  신발도매, 즉시연금 2017-17의 2개 조항, 즉시연금 2018-8 a·b 버전 각각), 그리고 정상
  문서의 24개 조항 중 23개(제19조 포함).
- **흔들린 조항**: 정상 문서 **제15조** 단 하나. risk_level이 "안전"(3/5, 60%)과
  "주의"(2/5, 40%) 사이에서 갈렸을 뿐 아니라, "주의"로 판정된 두 번조차 risk_type이 서로
  달랐다("책임 면제" 1회, "일방적 계약 해지" 1회) — **level도 type도 둘 다 흔들린 유일한
  사례**.

## 3. Judge 점수 편차 요약

Aspect별 표준편차를 문서마다 계산해 평균낸 값:

| Aspect | 평균 표준편차 | 비고 |
|---|---|---|
| clarity | **0.0** | 6개 문서 전부 5회 내내 5.0점으로 고정 — 전혀 흔들리지 않음 |
| faithfulness | 0.23 | 단일 조항 위약금 사례(광고대행, 신발도매)에서 가장 흔듦(±0.49) |
| risk_coverage | 0.32 | 신발도매에서 가장 흔듦(±0.63, 3~5점 사이 왕복) |
| actionability | 0.35 | 거의 모든 문서에서 고르게 ±0.4 안팎 흔들림 |

문서별 Judge 평균(4 aspect 평균) 자체의 최소~최대 폭은 문서마다 0.1~0.3점 사이였다(예:
신발도매 3.75~4.5, 즉시연금 2017-17 3.75~4.5). **가장 안정적인 aspect는 clarity, 가장
불안정한 aspect는 risk_coverage와 actionability**다 — 이는 "위험을 놓치지 않고 얼마나
빠짐없이 짚었는가"와 "행동 지침이 얼마나 구체적인가"라는, 서술 분량·디테일에 좌우되는
항목일수록 회차마다 LLM이 조금씩 다른 디테일을 넣거나 빼기 때문으로 보인다.

## 4. 정상 문서 오탐 변동

| 회차 | 오탐 개수 | 오탐 조항 |
|---|---|---|
| 1 | 1건 | 제19조 |
| 2 | **2건** | 제15조, 제19조 |
| 3 | 1건 | 제19조 |
| 4 | **2건** | 제15조, 제19조 |
| 5 | 1건 | 제19조 |

오탐 개수가 **1건과 2건 사이를 정확히 5회 중 3회(1건) / 2회(2건)로 오갔다** — 원인은
전부 제15조 하나다(제19조는 5회 내내 고정 오탐). 이는 이전 작업들에서 Ours 실행마다
오탐이 1건일 때도, 2건일 때도 있었던 관찰과 정확히 일치하는 재현이다.

## 5. retry_count(재생성 루프 발동) 기록

| 문서 | 1회차 | 2회차 | 3회차 | 4회차 | 5회차 |
|---|---|---|---|---|---|
| 광고대행 위약금 | 0 | 0 | 0 | 0 | 0 |
| 신발도매 위약금 | 0 | 0 | 0 | 0 | 0 |
| 즉시연금 2017-17 | 0 | 0 | 0 | 0 | 0 |
| 즉시연금 2018-8 (a) | 0 | 0 | 0 | 0 | 0 |
| 즉시연금 2018-8 (b) | 0 | 0 | 0 | 0 | 0 |
| 정상 예금약관 | 0 | 0 | 0 | 0 | 0 |

**30회 전부 retry_count = 0.** Judge 평균이 `JUDGE_THRESHOLD`(3.5점) 아래로 내려간 적이
한 번도 없어(최저 문서 평균 3.75점), 재생성 루프가 단 한 번도 발동하지 않았다. 즉 이번에
관찰된 모든 편차(제15조의 판정 흔들림, Judge 점수 ±0.1~0.6 흔들림)는 **재시도 로직과
무관하게, Analysis/Judge 각 노드의 LLM 호출 자체가 매번 조금씩 다른 응답을 낸 결과**다.

## 6. 토큰 사용량

| 문서 | 입력 토큰(5회 범위) | 출력 토큰(5회 범위) |
|---|---|---|
| 광고대행 위약금 | 3,286 ~ 3,677 | 1,450 ~ 1,912 |
| 신발도매 위약금 | 4,238 ~ 4,486 | 1,901 ~ 2,028 |
| 즉시연금 2017-17 | 7,807 ~ 8,078 | 2,698 ~ 3,006 |
| 즉시연금 2018-8 (a) | 3,464 ~ 3,654 | 1,554 ~ 1,962 |
| 즉시연금 2018-8 (b) | 7,077 ~ 7,515 | 2,594 ~ 3,127 |
| 정상 예금약관 | 80,465 ~ 81,862 | 18,443 ~ 19,185 |

**30회 합계**: 입력 538,918 토큰, 출력 149,020 토큰, 실행시간 합계 약 1,685초(28분).
문서당 토큰 사용량 자체는 회차 간 변동폭이 작다(정상 문서 기준 입력 토큰은
80,465~81,862로 ±1% 이내) — 즉 **입출력 토큰량의 재현성은 높고, 편차는 주로 판정
내용(risk_level/type/Judge 점수) 쪽에서 난다.**

## 7. 결론

- **위험 조항 판정(risk_level/risk_type) 자체는 재현성이 매우 높다** — 확정 위험 4건 +
  진단용 변형 1건, 총 7개 조항이 5회 내내 100% 동일했다.
- 유일하게 판정이 흔들린 곳은 정상 문서의 **제15조**(통지방법 및 효력) — 본인확인 없는
  제3자 통지의 유효성, 우송기간 경과 시 도달 간주 등 "면책성 절차 조항"들이 뒤섞여 있어
  경계선에 있는 애매한 조항으로 보인다. Judge 평균 점수는 문서마다 ±0.1~0.3점,
  aspect 기준으로는 clarity(±0.0, 고정)~risk_coverage/actionability(±0.3~0.35)까지
  흔들렸다.
- retry_count는 30회 전부 0이었다 — **재생성 루프는 이번 편차의 원인이 아니다.**
- **보고서에 수치를 쓸 때 감안할 편차**: risk_level/risk_type 판정 자체는 ±0으로 보고해도
  되지만(제15조류의 경계선 조항 제외), **Judge 평균 점수는 ±0.3점, 정상 문서 오탐
  건수는 "1~2건" 범위로 표기하는 것이 정확하다.**
